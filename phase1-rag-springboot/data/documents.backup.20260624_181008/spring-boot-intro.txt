Spring Boot 简介

Spring Boot 是一个基于 Spring 框架的快速开发平台，旨在简化 Spring 应用的初始搭建和开发过程。

核心特性：

1. 自动配置（Auto-Configuration）
Spring Boot 根据项目中添加的依赖自动配置应用。例如，如果你的类路径中有 spring-boot-starter-web，Spring Boot 会自动配置 Tomcat 和 Spring MVC。

自动配置的原理：
- 使用 @EnableAutoConfiguration 注解
- 通过 spring.factories 文件加载配置类
- 使用 @Conditional 注解实现条件化配置

2. 起步依赖（Starter Dependencies）
Spring Boot 提供了一系列"起步依赖"，简化 Maven 或 Gradle 配置。例如：
- spring-boot-starter-web: Web 应用开发
- spring-boot-starter-data-jpa: JPA 数据访问
- spring-boot-starter-security: 安全框架

3. 嵌入式服务器
Spring Boot 默认集成了嵌入式 Tomcat、Jetty 或 Undertow，无需单独部署 WAR 包。

4. Actuator
Spring Boot Actuator 提供生产级别的监控和管理功能，包括：
- 健康检查（/actuator/health）
- 指标收集（/actuator/metrics）
- 应用信息（/actuator/info）

5. 外部化配置
支持多种配置方式：
- application.properties 或 application.yml
- 环境变量
- 命令行参数
- @ConfigurationProperties 绑定

快速开始示例：

创建一个简单的 REST API：

```java
@RestController
@SpringBootApplication
public class HelloApplication {
    public static void main(String[] args) {
        SpringApplication.run(HelloApplication.class, args);
    }

    @GetMapping("/hello")
    public String hello() {
        return "Hello, Spring Boot!";
    }
}
```

常用注解：
- @SpringBootApplication: 组合注解，包含 @Configuration、@EnableAutoConfiguration、@ComponentScan
- @RestController: RESTful 控制器
- @Autowired: 自动注入依赖
- @Value: 注入配置值
- @ConfigurationProperties: 类型安全的配置绑定

最佳实践：
1. 使用 starter 依赖简化配置
2. 合理使用 profile 管理不同环境配置
3. 启用 Actuator 监控应用健康状态
4. 使用 Lombok 减少样板代码
5. 遵循 RESTful API 设计规范
