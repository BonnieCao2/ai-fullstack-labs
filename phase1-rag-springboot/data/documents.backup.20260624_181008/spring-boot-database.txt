Spring Boot 数据库配置指南

Spring Boot 提供了简化的数据库配置方式，支持多种数据源和 ORM 框架。

一、数据源配置

1. 添加依赖
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>
<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
</dependency>
```

2. 配置 application.yml
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb
    username: root
    password: password
    driver-class-name: com.mysql.cj.jdbc.Driver
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true
```

二、连接池配置

Spring Boot 默认使用 HikariCP 作为连接池（性能最优）。

配置示例：
```yaml
spring:
  datasource:
    hikari:
      maximum-pool-size: 10      # 最大连接数
      minimum-idle: 5            # 最小空闲连接数
      connection-timeout: 30000  # 连接超时时间（毫秒）
      idle-timeout: 600000       # 空闲超时时间
      max-lifetime: 1800000      # 连接最大存活时间
```

连接池调优建议：
- maximum-pool-size: CPU 核心数 * 2 + 磁盘数
- 根据实际负载调整，避免设置过大导致资源浪费
- 启用连接泄漏检测：leak-detection-threshold: 60000

三、JPA 实体定义

```java
@Entity
@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, unique = true)
    private String username;

    @Column(nullable = false)
    private String email;

    @CreatedDate
    private LocalDateTime createdAt;
}
```

四、Repository 层

```java
public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByUsername(String username);
    List<User> findByEmailContaining(String email);
}
```

Spring Data JPA 方法命名规则：
- findBy: 查询
- deleteBy: 删除
- countBy: 计数
- existsBy: 判断存在

五、事务管理

```java
@Service
public class UserService {
    @Transactional
    public void createUser(User user) {
        userRepository.save(user);
        // 如果后续操作失败，会自动回滚
    }
}
```

事务注解参数：
- propagation: 传播行为（REQUIRED、REQUIRES_NEW 等）
- isolation: 隔离级别（READ_COMMITTED、REPEATABLE_READ 等）
- rollbackFor: 指定回滚的异常类型
- timeout: 超时时间

六、多数据源配置

当需要连接多个数据库时：

```java
@Configuration
public class DataSourceConfig {
    @Bean
    @ConfigurationProperties("spring.datasource.primary")
    public DataSource primaryDataSource() {
        return DataSourceBuilder.create().build();
    }

    @Bean
    @ConfigurationProperties("spring.datasource.secondary")
    public DataSource secondaryDataSource() {
        return DataSourceBuilder.create().build();
    }
}
```

七、常见问题

1. 数据库连接池耗尽
原因：未正确关闭连接、连接泄漏
解决：启用连接泄漏检测、使用 try-with-resources

2. N+1 查询问题
原因：懒加载导致多次查询
解决：使用 @EntityGraph 或 JOIN FETCH

3. 事务不生效
原因：
- 方法不是 public
- 类内部调用（绕过代理）
- 异常被捕获未抛出

八、性能优化建议

1. 使用批量操作减少数据库往返
2. 合理使用索引
3. 启用二级缓存（Redis、Caffeine）
4. 使用连接池监控工具（如 HikariCP Metrics）
5. 避免 SELECT *，只查询需要的字段
